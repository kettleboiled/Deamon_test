# Глава 2: Динамические структуры данных

## 2.1 Проблема статических массивов

В первой части курса мы использовали массивы фиксированного размера (например, `int arr[100]`). У такого подхода есть два существенных недостатка:
1.  **Ограниченность размера:** Если данных окажется больше, чем 100, программа упадет или будет работать некорректно (Buffer Overflow).
2.  **Неэффективность вставки/удаления:** Чтобы вставить элемент в середину массива длины $N$, необходимо сдвинуть $N/2$ элементов вправо. Это операция $O(N)$, что неприемлемо медленно для больших объемов данных.

Решением являются **динамические структуры данных**, элементы которых разбросаны в памяти (в куче/heap) и связаны друг с другом через указатели.

## 2.2 Односвязный список (Singly Linked List)

Связный список состоит из узлов (`Node`). Каждый узел хранит полезные данные и указатель на следующий узел. Последний узел указывает на специальное значение `NULL`.

**Структура узла на языке C:**
```c
typedef struct Node {
    int value;           // Поле данных
    struct Node* next;   // Указатель на следующий элемент
} Node;
